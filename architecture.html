Livetutor — High-Level Architecture

Browser → FastAPI bridge → Gemini Live API. Native voice I/O, transcripts, and tool calls that render on a live whiteboard.

Browser layer Server layer (Python / FastAPI) Gemini Live API Binary audio (PCM) JSON control / tool calls

One PTT turn, end-to-end:

  1. User holds SPACE. Browser opens the mic, sends {type:"ptt_start"}. Server → session.send_realtime_input(activity_start=ActivityStart()).
  2. AudioWorklet emits 40 ms Int16 PCM chunks at 16 kHz. Each chunk is relayed as a binary WebSocket frame → server → send_realtime_input(audio=Blob(…)).
  3. User releases SPACE. Browser sends {type:"ptt_end"}. Server → send_realtime_input(activity_end=ActivityEnd()). Gemini begins the response.
  4. In parallel Gemini streams back: input_transcription (user STT) → user bubble; output_transcription (Sparks text) → assistant bubble; model_turn.parts[*].inline_data (24 kHz PCM) → browser playback queue; tool_call.function_calls[*] → browser tool dispatcher → KaTeX / function-plot / Rough.js renders.
  5. Browser acks each tool with {type:"tool_result", ok:true}; server has already sent an optimistic FunctionResponse back to Gemini.
  6. turn_complete closes both transcript bubbles and returns the avatar to idle.

Text-turn path (typed in input): {type:"text", text} → server → send_client_content(turns=Content(user, [Part(text)]), turn_complete=True). Gemini replies with audio + tools identically; the browser adds the user bubble locally since it already knows the text (no STT needed).
